# Rovia 产品设计文档

## 1. 产品概述

### 1.1 背景与痛点

- **专注力缺失**：ADHD 用户难以长时间维持对任务的关注。
- **空间感知弱**：用户容易在离开工作位后忘记未完成的任务。
- **情绪感知延迟**：用户往往在极度焦虑或压力过载后才察觉。
- **交互枯燥**：传统的待办清单缺乏情感反馈和即时激励。

### 1.2 产品愿景

通过“桌面宠物”这一具象化媒介，将冰冷的传感器数据转化为情感化的交互，构建一个能感知空间距离、生理压力并实时同步任务的闭环生态。

### 1.3 产品定位

Rovia 是一个面向 ADHD 使用场景的桌面专注支持系统。它不是单纯的 Todo 工具，也不是单纯的硬件仪表盘，而是一个由桌宠、手环、捏捏、摄像头和云端数据库共同组成的低压力陪伴式专注助手。

### 1.4 目标用户

- 有专注困难、任务切换困难、容易离桌后断线的 ADHD 用户
- 需要温和反馈而不是强提醒的学生和知识工作者
- 希望把生理状态、任务状态和环境状态放到同一个界面里管理的用户

## 2. 产品形态与系统组成

### 2.1 软件端

Rovia 当前的软件形态由四层组成：

1. **桌宠端（Electron 桌面应用）**
   - 常驻透明桌宠窗口
   - 桌宠上方小型摄像头预览框
   - 可展开的任务与数据面板

2. **桌面状态管理层**
   - 专注状态机
   - 本地持久化
   - Todo 管理
   - 传感器状态归一化

3. **硬件桥接层（Python BLE sidecar）**
   - 连接手环与捏捏两个蓝牙来源
   - 解析 HRV / 压力 / RSSI / 距离数据
   - 向桌面端广播标准化事件
   - 可选写入 Supabase

4. **云端数据层（Supabase）**
   - 账号登录与注册
   - `todos`、`focus_sessions`、`telemetry_data`、`app_events`
   - Realtime 同步

### 2.2 硬件端

- **智能手环**
  - 采集心率、RR interval、HRV、SDNN、压力评分等生理相关数据
  - 监控蓝牙 RSSI，推断与电脑的距离
  - 支持通过按键或通知触发 `enter_task`
  - 在 RSSI 低于阈值时接收提醒回写

- **捏捏设备**
  - 输出压力值或触发信号
  - 经过归一化和平滑后生成捏捏力度等级
  - 触发“捏捏脉冲”事件，统计频率
  - 可在高压力阈值下触发专注开始

- **电脑摄像头**
  - 当前已可被 Rovia 调用并显示实时预览
  - 当前阶段仅做本地预览，不做表情识别、姿态识别或视觉分析

## 3. 当前软硬件功能梳理

本节基于当前仓库实际实现状态整理，分为“已实现”“原型能力”“待接真实后端/算法”三类。

### 3.1 桌宠主窗口

**已实现**

- 透明背景流体桌宠形象
- 桌宠顶部小摄像头预览框
- 极简底部控制 dock
- 桌宠拖拽、归位、打开面板、开始专注、退出应用
- 20 分钟专注倒计时环
- 状态驱动的桌宠表现

**当前状态枚举**

- `Disconnected`
- `Idle`
- `Ready`
- `Support`
- `Focusing`
- `Away`
- `Completed`

### 3.2 面板信息架构

Rovia 当前面板已经形成多页结构：

- **专注页 Focus**
  - 当前会话
  - 今日轮次 / 时长
  - 身体信号
  - 7 日趋势
  - 最近专注记录

- **任务页 Todo**
  - 当前任务
  - 待办列表
  - 新增、改名、改标签、设为当前、完成

- **捏捏页 Squeeze**
  - 当前捏捏力度
  - 当前压力值
  - 最近 1 分钟频率
  - 最近 5 分钟频率
  - 最近一次捏捏时间
  - 10 分钟脉冲分布
  - 面向 ADHD 使用场景的调节提示文案

- **设备页 Device**
  - 生理状态、距离状态、同步状态
  - 传感器指标
  - 摄像头预览控制
  - 手动生理 / 距离预设

- **好友页 Friend**
  - 标签驱动的关系图和推荐好友视图
  - 当前仍偏前端探索原型，主要用于展示社交化专注方向

- **账号页 Account**
  - 登录 / 注册 / 退出登录
  - Demo 模式
  - 同步状态查看

### 3.3 任务与专注功能

**已实现**

- 本地 Todo 管理
- 当前任务与专注会话绑定
- 20 分钟专注开始、结束、重来一轮
- 离桌状态下切换为 `Away`
- 完成后生成最近一轮记录
- Focus 页展示最近会话与统计摘要

**与硬件联动的入口**

- 手环或捏捏发出 `enter_task` 时，可触发开始专注
- 离桌时，蓝牙距离可影响专注状态和提示语

### 3.4 生理与空间感知功能

**手环侧**

- 心率、RR interval、HRV、SDNN、focus score、stress score 的接入能力
- RSSI 轮询与距离估计
- 当 RSSI 低于阈值（如 `-60`）时触发提醒逻辑

**捏捏侧**

- 压力值解析
- 压力归一化与 EMA 平滑
- 力度等级划分：`idle / light / firm / squeeze`
- 由原始压力流生成“捏捏脉冲”事件
- 统计最近 1 分钟 / 5 分钟的使用频率

**电脑摄像头**

- 面板页和桌宠页都可显示摄像头实时预览
- 当前仅提供感知入口和 UI 呈现

### 3.5 云端与账号能力

**已实现**

- Supabase 邮箱密码登录
- Supabase 注册
- 本地 session 恢复
- Todo 同步
- `focus_sessions` 同步
- `telemetry_data` 写入能力
- `app_events` 事件写入能力

**当前主要数据表**

- `todos`
- `focus_sessions`
- `telemetry_data`
- `app_events`

### 3.6 好友与社交能力

**当前阶段定位**

好友页已经作为产品方向被纳入结构，但当前更接近“体验原型”而不是“真实社交系统”：

- 基于任务标签生成匹配关系
- 展示 Rovia 风格的关系图
- 支持“加好友 / 一起专注”的交互外观
- 还没有接入真实的好友关系表、邀请状态和多人同步后端

## 4. 软硬件闭环

Rovia 当前要构建的是一个“状态感知 -> 情感反馈 -> 任务承接 -> 云端同步”的闭环。

### 4.1 典型闭环 1：进入专注

1. 用户通过桌面按钮、手环触发或捏捏触发发起开始
2. Rovia 进入 `Focusing`
3. 当前任务被绑定到专注会话
4. 桌宠显示倒计时与状态变化
5. 专注记录同步到 `focus_sessions`

### 4.2 典型闭环 2：离桌提醒

1. 手环 RSSI 下降
2. sidecar 推断距离变远
3. Rovia 切换为 `Away`
4. 桌宠和面板给出温和提示
5. 若手环支持写回，则同时向硬件发送提醒

### 4.3 典型闭环 3：捏捏调节

1. 用户在焦虑、躁动或需要维持节奏时使用捏捏
2. sidecar 解析压力值并生成脉冲
3. Rovia 在“捏捏页”统计频率与力度
4. 桌宠根据状态给出非评判式反馈
5. 当频率或压力达到阈值时，可转为“进入一轮专注”

### 4.4 典型闭环 4：云端同步

1. 桌面端或硬件桥接层产生 Todo、专注、遥测事件
2. 数据写入 Supabase
3. 登录态恢复后，桌面端重新拉取并订阅
4. 面板显示最近记录、统计和同步状态

## 5. 面向 ADHD 的产品设计价值

### 5.1 低压力，而非高控制

Rovia 的设计重点不是监督用户，而是帮助用户在“要散掉之前”被温和接住。  
因此它采用的是：

- 桌宠陪伴，而不是冷冰冰的 KPI 看板
- 小步骤引导，而不是重型计划系统
- 频率、节奏、状态的反馈，而不是惩罚式提醒

### 5.2 多模态感知，而非单一 Todo

对 ADHD 用户来说，仅靠任务列表往往不足以支持执行。  
Rovia 的差异化在于把三类信号放到同一个产品里：

- **任务信号**：当前 Todo、完成进度、专注轮次
- **身体信号**：HRV、心率、压力、捏捏频率
- **空间信号**：蓝牙距离、离桌状态

### 5.3 情感化媒介，而非纯数据仪表盘

Rovia 的桌宠承担了两个角色：

- 将后台复杂状态转译成用户可感知的情绪化界面
- 降低用户面对压力数据时的心理负担

## 6. 当前实现状态评估

### 6.1 已具备的核心能力

- 桌宠桌面应用基础形态
- 任务管理与专注状态机
- 手环 / 捏捏 BLE sidecar 桥接
- Supabase 登录与核心表同步
- 摄像头调用与本地预览
- ADHD 场景下的捏捏频率页面

### 6.2 当前仍属原型或半实装的部分

- 好友页仍以体验原型为主
- 摄像头尚未接视觉算法
- 手环和捏捏的最终真实协议仍依赖设备 UUID / payload 对齐
- 部分生理与压力阈值仍需真实用户标定

### 6.3 当前产品成熟度判断

按产品形态看，Rovia 已经不是单页 demo，而是一个可运行的桌宠原型系统；  
按真实硬件联调与生产可用性看，仍处于“原型可演示、链路可扩展、待做真机打磨”的阶段。

## 7. 下一阶段产品建议

### 7.1 P0：把现有链路做稳

- 固定手环与捏捏的正式协议
- 稳定 BLE 重连和异常恢复
- 完善 `telemetry_data` 的真实写入与可视化

### 7.2 P1：让摄像头不只做预览

- 判断用户是否在桌前
- 检测离开 / 回归
- 作为辅助环境输入，而不是做侵入式监控

### 7.3 P1：让“捏捏页”更有干预意义

- 建立“频率异常 -> 降压建议 -> 轻任务推荐”的规则
- 将捏捏节奏与专注完成率做关联分析
- 支持“先调节，再进入 20 分钟任务”的引导

### 7.4 P2：补真实社交层

- 好友关系表
- 一起专注邀请与接受
- 同步中的多人状态可视化

## 8. 一句话总结

Rovia 当前的产品核心已经明确：  
它是一只放在电脑桌面上的情感化任务助手，通过手环、捏捏、摄像头与云端数据把“任务执行、情绪调节、空间感知”连接成一个更适合 ADHD 用户的低压力专注支持环境。
