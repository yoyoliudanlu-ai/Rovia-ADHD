## 1. 产品概述
### 1.1 背景与痛点
* **专注力缺失**：ADHD 用户难以长时间维持对任务的关注。
* **空间感知弱**：用户容易在离开工作位后忘记未完成的任务。
* **情绪感知延迟**：用户往往在极度焦虑或压力过载后才察觉。
* **交互枯燥**：传统的待办清单缺乏情感反馈和即时激励。

### 1.2 产品愿景
通过“桌面宠物”这一具象化媒介，将冰冷的传感器数据转化为情感化的交互，构建一个能感知空间距离、生理压力并实时同步任务的闭环生态。

---

## 2. 核心功能需求

### 2.1 电脑端：桌面宠物 (智能中转站)
* **F1: 蓝牙数据聚合**
    * 实时连接手环，获取 HRV (心率变异性) 与压力原始数据。
    * 利用蓝牙 RSSI 值计算用户与电脑的物理距离。
* **F2: 宠物行为逻辑 (状态机)**
    * **正常态**：用户在位且压力正常，宠物进行随机闲置动画。
    * **预警态**：`stress_level > 80`，宠物显示焦虑动画，引导用户深呼吸。
    * **休眠态**：`is_at_desk = false`，宠物进入睡眠状态以节省资源。
* **F3: 数据上报同步**
    * 每 10-15 秒将处理后的数据上报至 Supabase `telemetry_data` 表。

### 2.2 手机端：任务中心 (交互终端)
* **F4: 智能 TodoList**
    * **状态联动**：监听到用户“在位”时，App 自动置顶显示当前最高优先级任务。
    * **实时同步**：支持任务的增删改查，并同步至全端。
* **F5: 专注模式映射**
    * 当监听到 `focus_sessions` 有新记录（手环触发）时，手机端强制进入全屏倒计时界面。

### 2.3 手环端：物理触发 (硬件输入)
* **F6: 物理按键交互**
    * 用户点击手环物理键，触发电脑端插入一条专注计时记录。
* **F7: 震动反馈**
    * 当计时结束或压力连续过载时，手环触发物理震动提醒。

## 3. 数据库设计 (Supabase)

### 3.1 `telemetry_data` (实时遥测表)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `user_id` | UUID | 用户唯一标识 (49d91ddd-...) |
| `hrv` | Float | 心率变异性数值 |
| `stress_level` | Int | 0-100 压力分值 |
| `distance_meters` | Float | 蓝牙测距结果 |
| `is_at_desk` | Boolean | 逻辑判定用户是否在位 |

### 3.2 `todos` (待办任务表)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `task_text` | Text | 任务描述内容 |
| `is_completed` | Boolean | 是否已完成 |
| `priority` | Int | 优先级 (0:低, 1:中, 2:高) |

### 3.3 `focus_sessions` (专注会话表)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `start_time` | Timestamptz | 计时开始时间 |
| `duration` | Int | 预设时长 (分钟) |
| `status` | Text | running / completed / canceled |

## 4. 关键交互流程

### 4.1 自动专注流
1.  **物理触发**：用户按下手环物理键。
2.  **数据流转**：手环 --(BLE)--> 电脑 --(INSERT)--> Supabase。
3.  **实时广播**：Supabase --(Realtime)--> 手机端。
4.  **UI 响应**：手机端 UI 锁定，显示倒计时；桌面宠物戴上耳机，背景变为环境白噪音。

### 4.2 离位提醒流
1.  **物理触发**：用户起身离开（RSSI 减弱）。
2.  **逻辑判定**：距离 > 1.5m 且持续 10 秒，判定为离位。
3.  **状态同步**：Supabase 记录 `is_at_desk = false`。
4.  **手机反馈**：手机 App 推送消息：“别忘了刚才还没做完的 [任务名] 哦，宠物在等你回来。”

---

## 5. 非功能性需求
* **安全性**：所有表必须启用 RLS (Row Level Security)，确保隐私数据仅用户本人可见。
* **实时性**：Realtime 通讯延迟需控制在 1 秒以内。
* **容错性**：蓝牙连接中断时，桌面宠物需显示“离线”UI，并停止上报错误测距。
